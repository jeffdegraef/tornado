FAQ for Developers

How is Node-RED embedded?

We follow the documentation here:

http://nodered.org/docs/embedding.html

Where do I find the NodeRED settings ?

In the file root\thethingbox\thethingbox.js

The settings are there:

var settings_nodered = { // http://nodered.org/docs/configuration.html
    httpAdminRoot:"/",
    httpNodeRoot: "/api/",
    userDir: userDir,
    flowDir: flowDir, // TTB specific
    flowFile: flowDir + "flows.json",
    systemFlowDir: "/root/thethingbox/node_modules/node-red/flow",
    verbose: true,
    flowFilePretty: true,
    otherPropToSave: ['origin'], // properties to keep when dealing with nodes in memory
    paletteCategories:['subflows', 'input', 'output', 'function', 'social', 'storage', 'analysis', 'advanced'],
    hiddenWorkspaces: ['system'],
    functionGlobalContext: {
       settings: settings_ttb
       ,userDir: userDir
       ,settingslib: require('/root/thethingbox/settingsttb.js')
       ,wpi: require('node-red/node_modules/wiring-pi') // https://www.npmjs.com/package/wiring-pi
       ,CryptoJS: require('node-red/node_modules/crypto-js') // https://www.npmjs.com/package/cryptojs
    }
};

Note: the code above is just an example. It was not not updated since pasted here.

How can I find its ip address ?

Connect via SSH on your thethingbox. Enter : ifconfig and the ip will be written like that :

eth0 
inet addr: 192.168.0.21

How to launch NodeRED manually?

From SSH: systemctl stop thethingbox.service cd thethingbox node thethingbox.js

how to npm install new nodes

Many nodes can be install with the “import / node” UI.

But you can add by yourself.

If npm install is done where package.json of nodered is, it should work. try here: root\thethingbox\node_modules\node-red

The Thingbox expands the SDCard at first boot

At first boot, a Thingbox expands to the whole SDCard with resize2fs Script is in root.auto-expand-scripts (deleted at first boot)

How Node-RED is integrated

NodeRED is run following the “Embedding into an existing app” as described in the Node-RED documentation here.

Where are my datas?

Flows and settings are stored outside in /root/userdir Be aware that if you install additional nodes yourself, they may store their data elsewhere.

Where is the flow stored?

Starting with the version 1.8.0, the used flow is the dynamic merge of

  • The user flow in \root\userdir\flows.json
  • The Thingbox flows in \root\thethingbox\node_modules\node-red\flow\
  • flows in nodes

See here.

The log is sent to the console.

The console log is sent to console.txt in the public folder so it can be read by the client browser. There is a menu item in the NodeRED right menu that opens it.

/root/thethingbox/node_modules/node-red/public/console.txt

Edit the PORT var in \root\thethingbox\thethingbox.js

I need to add an SSL certificate to thethingbox to do https calls.

you can find info at

http://nodered.org/docs/configuration.html
https://github.com/node-red/node-red/blob/master/settings.js

In particular:

// The following property can be used to enable HTTPS
// See http://nodejs.org/api/https.html#https_https_createserver_options_requestlistener
// for details on its contents.
// See the comment at the top of this file on how to load the `fs` module used by
// this setting.
//
//https: {
//    key: fs.readFileSync('privatekey.pem'),
//    cert: fs.readFileSync('certificate.pem')
//},

The settings are in the file root\thethingbox\thethingbox.js

Generate private key and certificate for https with nodejs

Create a new directory and install npm module pem

$ mkdir generatepem
$ cd generatepem
$ npm i pem

Edit a file index.js with this content:

var fs = require("fs");
var pem = require("pem");

pem.createCertificate({days: 365, selfSigned: true}, function(err, keys) {

    fs.writeFile("privatekey.pem", keys.serviceKey);
    fs.writeFile("certificate.pem", keys.certificate);

});

Generate the files:

$ node index.js

You’ve got now privatekey.pem and certificate.pem into the generatepem directory. You can copy now copy/cut paste anywhere you need !
